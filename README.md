# Mass Finding

Mass Finding 是一个面向质谱分析场景的桌面软件，主要用于根据质荷比信息推测分子量、生成候选分子式，并结合 PubChem 数据库进行化合物检索与结果整理。软件以 Windows 桌面使用为主要目标，界面直观，适合科研、教学和日常数据辅助分析。

---

## 一、软件定位

在实际质谱分析中，研究者通常会遇到以下问题：

- 已知某个峰的 m/z，但不确定真实分子量；
- 想在误差范围内快速列出所有可能的分子式组合；
- 希望进一步查询候选分子式对应的 PubChem 化合物信息；
- 希望保存本地缓存，避免重复查询与重复计算。

Mass Finding 的设计目标，就是把这些步骤集中到一个统一的图形化工具中完成，减少人工筛选与重复操作成本。

---

## 二、核心功能

### 1. 质谱分子量换算
软件支持根据常见质谱模式与离子加合类型，对输入的 m/z 值进行换算，得到候选分子量范围。

支持的分析条件包括：
- MS 模式选择，如 ESI+、ESI- 等；
- 电荷数设置；
- 百分比误差与绝对误差双模式控制；
- 常见加合离子参与换算。

这使得用户可以从实验峰位快速反推出更贴近真实情况的分子量区间。

### 2. 候选分子式生成
在给定目标分子量、误差范围、元素组成约束后，软件可以自动生成所有满足条件的候选分子式。

功能特点：
- 支持多元素范围限制；
- 自动考虑化合价规律与不饱和度；
- 自动计算候选式的分子量与相关属性；
- 支持较大规模搜索任务；
- 内部已针对多进程计算场景进行了适配，便于后续 exe 发布使用。

### 3. PubChem 联网检索
软件可以根据分子式自动查询 PubChem 数据库，获取候选化合物记录，并进行一定的排序整理。

检索结果通常可包含：
- CID；
- 化合物名称；
- IUPAC 名称；
- 分子量；
- 单同位素质量；
- 同义词信息；
- 部分结构相关字段。

### 4. 本地缓存与断点复用
为减少重复访问网络接口，软件内置了缓存机制，会自动保存：
- 分子式生成结果；
- PubChem 原始数据；
- 初始化索引信息；
- 日志与结构预览相关缓存。

这样即使重复查询同一分子式，也能更快返回已有结果。对于网络不稳定或受限环境，本地缓存也能提供一定的降级使用能力。

### 5. 图形化交互界面
软件基于桌面 GUI 实现，提供清晰的功能区划分，包括：
- 导航栏；
- 功能操作区；
- 分子式结果展示区；
- 状态栏与日志区域。

用户无需编写命令行指令，即可完成主要分析流程。

## 三、典型使用流程

推荐按以下顺序使用本软件：

### 步骤 1：输入质谱参数
在分子量分析页面输入：
- m/z；
- 误差范围；
- 电荷数；
- 离子模式；
- 元素上下限条件。

### 步骤 2：生成候选分子式
点击分析后，软件会计算符合条件的候选式，并展示对应结果。

### 步骤 3：加入待搜索列表
将感兴趣的分子式加入待搜索区域，便于进一步做 PubChem 检索。

### 步骤 4：检索化合物信息
软件会尝试从 PubChem 获取匹配记录，并在成功、失败、部分成功等状态之间做分类展示。

### 步骤 5：复核与导出
用户可以结合名称、质量、同义词等信息判断候选化合物，并将结果导出或作为后续实验参考。

## 九、缓存说明

软件会在本地自动创建缓存目录，用于存储：
- 已生成的分子式结果；
- PubChem 原始查询数据；
- 初始化数据索引；
- 日志与结构预览缓存。

缓存的意义：
- 提升重复查询速度；
- 降低网络请求次数；
- 为离线或弱网环境提供一定支持。

如果需要彻底清理缓存，可删除本地对应缓存目录后重新启动软件。

## 十、联网说明

PubChem 检索功能依赖外部网络。如果用户处于以下环境，联网结果可能受到影响：
- 无外网；
- 防火墙限制；
- 代理未配置；
- PubChem 服务临时波动。

在这类情况下：
- 新查询可能失败；
- 已缓存数据仍可能继续使用；
- 软件会尽量保留部分结果与失败信息，便于后续重试。

## 十一、常见问题

### 1. 为什么第一次启动 exe 比较慢？
单文件 exe 首次启动时需要解包临时资源，因此会比源码运行稍慢，这是正常现象。

### 2. 为什么 PubChem 搜索没有结果？
可能原因包括：
- 当前分子式本身无匹配记录；
- 网络受限；
- 接口响应超时；
- 只命中了部分缓存或部分批次失败。

### 3. 软件会修改原始数据吗？
软件主要会在本地生成缓存、日志和导出文件，不会主动修改你的实验原始文件。

### 4. 能否离线使用？
基础公式生成与本地缓存读取可以继续使用，但在线 PubChem 检索依赖网络。
